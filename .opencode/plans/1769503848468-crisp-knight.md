# Plan: Improve Mobile Responsiveness

## Problem Summary

From the screenshot, lyrics lines are getting cut off on mobile devices. The line "I used to ship but now that build don't feel ri..." is truncated on the right edge.

## Root Cause

In `src/components/LyricsDisplay.tsx:99`, the current line gets `scale: 1.15` applied via Framer Motion. This 15% width increase causes longer lines to overflow the container on narrow mobile screens.

Current code:
```tsx
const scale = isCurrent ? 1.15 : 1;
```

## Solution

### Changes to `src/components/LyricsDisplay.tsx`

1. **Add `useMediaQuery` hook** to detect mobile viewport
2. **Reduce scale on mobile**: `1.05` on mobile, `1.15` on desktop (sm+: 640px)
3. **Reduce horizontal padding**: `px-2 sm:px-4` (8px mobile, 16px desktop)
4. **Add `break-words`** class as safety net for very long words
5. **Reduce base text size** slightly: `text-sm sm:text-base md:text-lg lg:text-xl`

### Implementation Details

**Step 1**: Create a simple `useMediaQuery` hook or use `window.matchMedia` inline:

```tsx
const [isMobile, setIsMobile] = useState(false);

useEffect(() => {
  const mq = window.matchMedia("(max-width: 639px)");
  setIsMobile(mq.matches);
  const handler = (e: MediaQueryListEvent) => setIsMobile(e.matches);
  mq.addEventListener("change", handler);
  return () => mq.removeEventListener("change", handler);
}, []);
```

**Step 2**: Pass `isMobile` to `LyricLineComponent` and use it:

```tsx
const scale = isCurrent ? (isMobile ? 1.05 : 1.15) : 1;
```

**Step 3**: Update container classes:
- From: `px-4`
- To: `px-2 sm:px-4`

**Step 4**: Update line classes:
- From: `text-base sm:text-lg md:text-xl lg:text-2xl`
- To: `text-sm sm:text-base md:text-lg lg:text-xl break-words`
